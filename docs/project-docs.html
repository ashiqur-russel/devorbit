1 · Overview

// what and why

What is DevOrbit?

DevOrbit gives students and small teams a single dark-mode dashboard to monitor everything that matters in their DevOps workflow — GitHub Actions pipeline runs, server CPU/RAM/disk in real time, and a registry of every deployed application. Instead of logging into five different services, DevOrbit pulls data from GitHub, GitLab, and Vercel and streams live server metrics from a zero-dependency Node.js agent running on your hardware.

Pipeline Tracking

Server Health

A lightweight Node.js agent runs on your server and streams CPU, RAM, disk, and network metrics every 5 seconds via WebSocket. Live charts update without polling.

Deployment Registry

Track every Vercel (and later Docker, AWS, OVH) deployment: platform, URL, status, and timestamp. The Vercel poller syncs automatically on integration connect.

Auth & Teams

GitHub OAuth login — no passwords. Users belong to team workspaces and can invite members. All pipelines, servers, projects, and integrations are scoped to a team.

WebSocket flow: The agent connects to the Socket.io gateway using an agentToken. The gateway validates the token, marks the server online, and forwards each metric event to the server:{serverId} room. The browser dashboard subscribes to that room and updates charts in real time without any polling.

7 · Quick Start

// docker compose — up in 3 minutes
1

Create GitHub OAuth App

Go to github.com → Settings → Developer Settings → OAuth Apps → New OAuth App.

Application name: DevOrbit Homepage URL: http://localhost:3000 Callback URL: http://localhost:4000/api/v1/auth/github/callback

After registering, copy the Client ID and generate a Client Secret.

2

Configure environment

# clone and copy env git clone https://github.com/ashiqur-russel/devorbit.git cd devorbit cp .env.example .env

Fill in .env:

JWT_SECRET=any-long-random-string-change-me GITHUB_CLIENT_ID=Ov23liXXXXXXXXXX GITHUB_CLIENT_SECRET=your-secret-here GITHUB_CALLBACK_URL=http://localhost:4000/api/v1/auth/github/callback WEB_URL=http://localhost:3000 API_URL=http://localhost:4000 WS_URL=ws://localhost:4000
3

Start everything

docker compose up -d # starts MongoDB, Redis, API, and Web # open http://localhost:3000 → sign in with GitHub
4

Run the monitoring agent on any server

Get your agent token from Settings → Agent Setup in the dashboard, then on your server:

npx @devorbit/agent start \ --token=your-agent-token \ --api=http://localhost:4000 # metrics stream to the dashboard within 5 seconds
5

Connect integrations

Go to Settings → Integrations in the dashboard. Paste a GitHub Personal Access Token (needs repo + workflow read scopes) and a Vercel API token. DevOrbit will start polling and sync data within 5 minutes.

10 · Security Considerations

// threat model for self-hosters
🔑

Agent Tokens

Each server gets a unique signed JWT as its agent token. Tokens are scoped to a single server and can be revoked by deleting the server record.

🔒

Integration Tokens

Provider API tokens (GitHub, Vercel) are stored in MongoDB. In production, apply AES-256 encryption at the application layer before persisting.

🛡

WebSocket Auth

The Socket.io gateway validates agentToken on connection. Unauthenticated sockets are disconnected before any events are processed.

⚡

Rate Limiting

Redis-backed per-IP and per-team rate limiting on all API routes. Auth endpoints are additionally throttled to prevent brute-force attacks.

🌐

CORS

Strict origin whitelist — only WEB_URL is allowed as CORS origin. Credentials are enabled for cookie-based session support.

🔐

HTTPS

Caddy reverse proxy provides automatic HTTPS with Let's Encrypt for self-hosters. Never expose the API port directly in production.
