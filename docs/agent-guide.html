The DevOrbit Agent is a lightweight Node.js process that runs on your server and streams CPU, RAM, disk, and network metrics to your DevOrbit dashboard every 5 seconds. Zero dependencies on the server — just Node.js.

devorbit-agent
Node.js 18+
WebSocket
5 seconds

1 · How It Works

// read-only, zero impact on your server
📡

Collects Metrics

Uses the systeminformation npm package to read CPU load, RAM usage, disk usage, and network I/O from the OS every 5 seconds.

⚡

Streams via WebSocket

Connects to your DevOrbit API via Socket.io using the agent token as authentication. Sends a metric event every 5 seconds. Auto-reconnects on disconnect.

🛡

Read-Only — Safe

The agent only reads system stats. It does not open any ports, does not modify files, and has no impact on running processes or server performance.

3 · Install & Run

// one command on any server
1

Get your agent token

In your DevOrbit dashboard, go to Settings → Agent Setup. Click Register Server, give it a name, and copy the generated token.

2

SSH into your server

ssh ubuntu@your-server-ip
3

Start the agent

Run directly with npx — no global install needed:

npx devorbit-agent start \ --token=your-agent-token-here \ --api=http://your-devorbit-api:4000

You should see:

DevOrbit Agent v0.1.0 Connecting to http://your-devorbit-api:4000... ✓ Connected (socket: abc123)
4

Verify in dashboard

Open DevOrbit → Servers → click your server → live CPU/RAM charts should appear within 5 seconds.

// survive SSH disconnect and server reboots
Why PM2? Running npx directly stops when you close SSH. PM2 keeps the agent running in the background, restarts it if it crashes, and auto-starts it on server reboot.

5 · Managing the Agent

// start · stop · restart · logs
Check status
pm2 list # shows all running processes pm2 show devorbit-agent # detailed agent info ps aux | grep devorbit # if not using pm2
Stop the agent
pm2 stop devorbit-agent # stop but keep in pm2 list pm2 delete devorbit-agent # stop and remove completely pkill -f "devorbit-agent" # force kill if not using pm2
Restart & reload
pm2 restart devorbit-agent # restart the agent pm2 reload devorbit-agent # zero-downtime reload
View logs
pm2 logs devorbit-agent # live log stream pm2 logs devorbit-agent --lines 50 # last 50 lines pm2 flush # clear all logs

6 · Token Management

// one token per server · keep them secret
⚠ Never share your agent token publicly. It gives access to stream data to your dashboard. If exposed (e.g. in a chat, screenshot, or git commit), rotate it immediately — delete the server in the dashboard and register a new one.
Token type
Purpose
Scope
Agent token
Authenticates the agent to stream metrics to your dashboard. Generated per server in Settings → Agent Setup.
Per server
JWT (user)
Your login session token stored in the browser. Never use this as an agent token.
User session
!

How to rotate a compromised token

1. Go to DevOrbit dashboard → Servers → find the server → click Delete

2. Go to Settings → Agent Setup → Register Server → copy the new token

3. On your server, restart the agent with the new token:

pm2 delete devorbit-agent pm2 start "npx devorbit-agent start --token=NEW_TOKEN --api=http://your-api:4000" --name devorbit-agent pm2 save

7 · Troubleshooting

// common issues and fixes
❌ Connection error: connect ECONNREFUSED

The API URL is wrong or the API server is not running.

Fix: Check that your DevOrbit API is running and accessible. Test with:

curl http://your-api:4000/api/v1/auth/profile # should return 401, not connection refused
❌ Error: --token is required

You forgot to pass the --token flag or misspelled it.

# Wrong npx devorbit-agent start token=abc123 # Correct npx devorbit-agent start --token=abc123
❌ Agent connects but no metrics in dashboard

The agent is connected but the dashboard is not subscribed to the right server room.

Fix: Make sure you are viewing the correct server in the dashboard. The server ID in the URL must match the token's server ID. Try refreshing the page.

❌ Agent stops when I close SSH

You ran it directly without PM2. Use PM2 to keep it running in the background.

npm install -g pm2 pm2 start "npx devorbit-agent start --token=TOKEN --api=URL" --name devorbit-agent pm2 save
❌ npx: command not found

Node.js is not installed on the server.

# Ubuntu / Debian curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash - sudo apt-get install -y nodejs # Verify node --version # should be v20.x npm --version # should be 10.x+
❌ Token expired or invalid

Agent tokens are JWTs with an expiry. If the token has expired, generate a new one.

Fix: Dashboard → Servers → delete server → Settings → Agent Setup → Register Server → copy new token → restart agent with new token.
